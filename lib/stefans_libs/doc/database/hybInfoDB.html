NAME

database::designDB

DESCRIPTION

This class is a MySQL wrapper that is used to access the table HybInfo where all the different hybridization descriptions are stored.

Depends on

the ::NimbleGene_config manpage

the ::root manpage

the database::antibodyDB manpage

the database::cellTypeDB manpage

the database::designDB manpage

CreateDB

Creates a new table to store the hybridization information foreach array hybridization used. This method automatically deleted all stored information in a old table!

arguments

none

atributes

[0]: the NimbleGene Array Id

[1]: the DNA labeling dye (cy3 or cy5) or the antibody specificity string

retrun value

true if the hybridization is found in the table or false if it is not found

GetInfoID_forHybType

Each array hybridization contains two diferent DNA samples. Each of these samples has its own hybridization id, but this method returns only one of these two. Use this method if you need only one id ,for example to store the enrichment factors for one 'ChIP on chip' experiment.

atributes

See GetInfoIDs_forHybType

GetInfoIDs_forHybType

atributes

[0]: the antibody string as accepted by SelectId_BySpecificity in the database::antibodyDB manpage

[1]: the celltype string as accepted by SelectID_ByCellType in the database::cellTypeDB manpage

[2]: the organism string as accepted by SelectID_ByCellType in the database::cellTypeDB manpage

[3]: the design string as accepted by SelectId_ByArrayDesignString in the database::designDB manpage

return value

the reference to a array containing the HybInfo table line id's matching to the search parameters

insertData

atributes

[0]: the NimbleGene id

[1]: the DNA labeling marker (cy3 or cy5)

[2]: the antibody specificity as accepted by SelectId_BySpecificity in the database::antibodyDB manpage

[3]: the celltype as accepted by SelectID_ByCellType in the database::cellTypeDB manpage

[4]: the organism string as accepted by SelectID_ByCellType in the database::cellTypeDB manpage

[5]: the template type (experimental or input) as defined in the NimbleGene SampleKey.txt file

[6]: the design string as accepted by SelectId_ByArrayDesignString in the database::designDB manpage

return values

true if the array hybridization was found in the table after insertion, false if not

SelectHybInfo_ByID

atributes

a hashref with the sturcture {ID => the intername HybInfo table line id, NimbleGen_Id => the NimbleGene 'Chip on chip' experiment id, Marker => the DNA labeling color (cy3 or cy5), TemplateType => the template type as defined in the NimbleGene SampleKey.txt file, Celltype_ID => as returned from SelectID_ByCellType in the database::cellTypeDB manpage, Antibody_ID => as returned from SelectId_BySpecificity in the database::antibodyDB manpage, ArrayDesignId => as returned from SelectId_ByArrayDesignString in the database::designDB manpage } =cut

sub SelectHybInfo_ByID { my ( $self, $ID ) = @_; # print ``SelectHybInfo_ByID $ID\n''; return $self->{usedData_byID}->{$ID} if ( defined $self->{usedData_byID}->{$ID} ); if ( $self->DataExists($ID) ) { return $self->{usedData_byID}->{$ID}; } return undef; }
